The new filters allow you to find the exact item you want.

Now you can add multiple "groups" of explicit mod searches. Every group can have its own operator:

  • And: This is what previous form used to do all the time. All mods must be present on an item and all mods must match the specified values.
  • Not: None of specified mods must exist. The value field is ignored.
  • Count: The specified number (on the right) of mods must match the specified values. Note that searching for "count" and setting the "min" field to zero makes no sense.
  • Sum: All specified mods' values are summed and the final value is checked against the total value (on the right).
